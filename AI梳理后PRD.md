> 项目名称：智慧医院数字孪生 IOC
> 文档版本：v1.0

## 一、原始需求摘要

基于医院场景的 BIM 数字孪生平台，包含：

- **综合管理**：医院全景展示、左/右侧数据面板、楼宇 POI、空间管理（导览/拆楼/房间ROOM）
- **设备管理**：设备查询、新增（床位/仪器）、迁移、升级、卸载

---

## 二、技术域分类

| 优先级 | 技术域                               | 对应 Skill                             |
| ------ | ------------------------------------ | -------------------------------------- |
| 必须   | 场景初始化与接入                     | `wdp-api-initialization-unified`     |
| 必须   | BIM 拆楼/楼层控制                    | `wdp-api-bim-unified`                |
| 必须   | 相机漫游/聚焦                        | `wdp-api-camera-unified`             |
| 必须   | 实体操作（显隐/高亮/移动/替换/删除） | `wdp-api-entity-general-behavior`    |
| 必须   | 覆盖物（楼宇 POI 标签）              | `wdp-api-coverings-unified`          |
| 必须   | 图层/模型显隐控制                    | `wdp-api-layer-models`               |
| 必须   | 事件注册（点击选中）                 | `wdp-api-general-event-registration` |
| 次要   | 属性操作（实体属性读写）             | `wdp-api-generic-base-attributes`    |
| 次要   | 材质设置（房间高亮着色）             | `wdp-api-material-settings`          |

## 四、功能模块拆解

### 模块 A：综合管理（首页）

#### A-1 初始化

- **触发时机**：页面加载完成，SDK 就绪后
- **操作**：
  1. 初始化 WdpApi 实例（BIM 模式）
  2. 等待 `progress === 100` 场景完全加载
  3. 镜头飞至全景视角（坐标见 3.2）
  4. 在场景中自动开启**楼宇分布 POI**（门诊楼、住院楼两个 POI 点）
  5. 渲染左侧统计面板（空间管理、综合安全管理、医疗设备管理）
  6. 渲染右侧统计面板（能源管理、电梯管理、后勤服务管理）

#### A-2 楼宇分布 POI

- **数据**：门诊楼、住院楼（坐标待场景确认）
- **样式**：覆盖物标签，悬浮于楼宇上方
- **交互**：点击 POI 可聚焦对应楼宇

### 模块 B：空间管理

#### B-1 环境导览

- **触发**：点击"环境导览"按钮
- **API**：相机漫游（`wdp-api-camera-unified`）
- **路径**：6 个路径点（见 3.3），每段 20s，循环
- **其他**：其余相机参数使用默认值

#### B-2 楼宇拆解（门诊楼）

- 分层展开（Z 轴偏移）：
  - F4：Z 轴 + 30

- 左侧展示楼宇总览数据面板
- 弹出 F1\~F4 楼层切换按钮

**Step 2 — 点击楼层按钮（F1\~F4）：**

- 其他层**隐藏**
- 当前层执行 **BIM 拆楼 API**（`wdp-api-bim-unified`）

### 模块 C：设备管理

#### C-0 进入设备管理模块

- 点击"设备管理"入口
- 镜头聚焦 **F3 楼层**
- 场景**隐藏其余层模型**（保留F3显示）
- **锁定 F4、Roof**（防止鼠标点选穿透，避免无法选中设备）

#### C-1 医疗设备总览（数据面板）

- 展示设备总数、各类设备数量、能耗汇总、设备效益等统计指标
- 详见原型设计

#### C-2 设备查询

**方式1：搜索框模糊查询**

- 输入设备名称关键字
- 高亮匹配设备（`wdp-api-entity-general-behavior`）
- 镜头聚焦至设备位置
- 右侧弹出设备信息详情

**方式2：点击场景中设备**

- 注册实体点击事件
- 触发后：高亮该设备 + 右侧弹出信息详情 + 镜头聚焦

**设备详情展示字段**：名称、所处位置、安装日期、运维日期、设备能耗、设备效益、使用状态

#### C-3 新增设备

**业务场景1：新增床位**

- 点击"新增床位"按钮
- 弹框展示可新增床位的房间列表
- 选择房间后：
  - 在对应房间内新增床位模型（预置模型显隐+位置设置）
  - 镜头聚焦至该房间
  - 台账中写入新设备记录（安装日期=操作时间）

**业务场景2：新增仪器**

- 点击"新增仪器"按钮
- 弹框展示可新增仪器的房间及仪器类型
- 选择房间及仪器类型后：
  - 在对应房间内新增仪器模型（预置模型显隐+位置设置）
  - 镜头聚焦至该房间

#### C-4 设备迁移

- 在场景中选中设备 → 点击"设备迁移"
- 弹框展示可迁移房间列表
- 选择目标房间后：
  - 将设备模型移动至目标房间坐标（`wdp-api-entity-general-behavior` 移动 API）
  - 镜头聚焦至目标房间
  - 台账中更新设备所处位置

#### C-5 设备升级

- 在场景中选中设备 → 点击"设备升级"
- 弹框展示可升级的设备型号列表
- 选择升级型号后：
  - **隐藏原设备模型**
  - **显示升级后预置模型**（同位置，通过 Eid 控制显隐）
  - 台账中更新设备名称和 Eid

#### C-6 设备卸载

- 在场景中选中设备 → 点击"设备卸载"
- 弹框确认"是否确认卸载？"
- 确认后：
  - **从场景中隐藏（或删除）该设备模型**
  - 台账中将该设备标记为已卸载

## 六、技术依赖关系

### 6.1 技术依赖

| 依赖组件            | 用途               |
| ------------------- | ------------------ |
| `CloudApi.min.js` | 底层渲染引擎       |
| `wdpApi.min.js`   | WDP 主 API         |
| `BimApi`（插件）  | BIM 拆楼、楼层控制 |

### 6.2 时序依赖

```
加载 SDK
  └─> 创建 WdpApi 实例
        └─> 启动渲染器
              └─> 等待 onProgress(100) / onReady
                    ├─> 安装 BIM 插件
                    ├─> 镜头飞至全景
                    ├─> 开启楼宇 POI
                    └─> 渲染左右统计面板
                          ├─> [综合管理] 注册交互事件
                          ├─> [空间管理] 楼宇拆解 / 漫游 / 房间管理
                          └─> [设备管理] 显隐楼层 + 设备增删改查
```

### 6.3 数据依赖

```
台账数据（JS 常量/JSON 文件）
  ├─> 楼宇管理表 → 楼层面板数据展示
  ├─> 房间管理表 → 房间高亮 + 信息面板
  └─> 设备管理表 → 设备查询 / 新增 / 迁移 / 升级 / 卸载
```

## 七、执行计划（代码实现路径）

### 阶段 1：初始化

- 搭建 HTML 基础骨架（参考 `universal-bootstrap.template.html`）
- 配置 SDK 引入（CloudApi + wdpApi）
- 创建 WdpApi 实例，绑定渲染容器 `#player`
- 注册场景加载进度回调 / 就绪回调

### 阶段 2：准备阶段

- 安装 BIM 插件
- 场景就绪后飞至全景镜头
- 初始化数据层（楼宇台账、房间台账、设备台账）

### 阶段 3：综合管理首页

- 渲染左/右侧统计面板（纯 HTML/CSS）
- 创建楼宇 POI 覆盖物（门诊楼、住院楼）

### 阶段 4：空间管理模块

- 实现环境导览（相机路径漫游）
- 实现楼宇拆解（Layer 显隐 + Z轴偏移 + BIM 拆楼）
- 实现房间管理（类型筛选 + 房间高亮 + 信息面板）

### 阶段 5：设备管理模块

- 实现设备总览数据面板
- 实现设备查询（搜索框 + 场景点击）
- 实现设备新增（预置模型显隐 + 镜头聚焦）
- 实现设备迁移（位置移动 + 台账更新）
- 实现设备升级（模型替换）
- 实现设备卸载（模型隐藏 + 台账标记）

### 阶段 6：验证与收尾

- 测试各功能模块交互闭环
- 错误处理与边界情况处理
- 代码注释与整理

## 九、技术风险与注意事项

1. **BIM 拆楼与楼层显隐**：需要安装并正确使用 BIM 插件，必须确认 F4、Roof 锁定逻辑，否则设备管理鼠标点选会穿透误选。
2. **设备新增/升级**：通过预置隐藏模型显隐实现，所有预置模型的 Eid 已在台账中维护，注意初始化时需批量隐藏这些预置模型。
3. **场景就绪时序**：所有业务操作必须在 `progress === 100` 或 `onReady` 回调后执行，避免场景未加载完成时操作失败。
4. **相机漫游停止机制**：环境导览启动后需提供"停止漫游"按钮，避免漫游过程中无法切换其他功能模块。
